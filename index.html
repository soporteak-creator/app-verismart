AWS Lightsail: recursos al límite

El bundle de $10/mes entrega solo 2 GB RAM y 1 vCPU. Con Vue SSR + Node.js + MongoDB corriendo en el mismo host via Docker Compose, el servidor opera constantemente por encima del 85% de memoria, causando lentitud y caídas.

MongoDB Atlas M0: al límite de almacenamiento

El tier gratuito M0 permite solo 512 MB de almacenamiento. La base de datos ya alcanzó el 90% de ese límite. El siguiente tier (M10) cuesta $57/mes adicionales — un salto de precio completamente desproporcionado.

Redundancia

Docker Compose en host único: sin resiliencia

Todos los servicios corren en un solo Docker Compose en una única instancia. Si esa instancia falla por cualquier motivo, toda la plataforma cae. No existe load balancing, auto-scaling ni failover automático.

Alternativas

¿Qué opciones tenemos?

Comparamos las principales plataformas de hosting modernas con soporte Docker nativo, evaluando costo, performance y facilidad de migración.

Métrica AWS Lightsail Railway Render Fly.io DO App Platform
🏆
Recomendación: Railway

Railway ofrece el mejor balance entre costo y rendimiento para el stack de VeriSmart. Deploy automático desde GitHub, soporte nativo de Docker Compose, plugin de MongoDB con 1 GB incluido (vs 512 MB en Atlas M0), y 8 GB RAM por ~$20/mes. La migración desde AWS Lightsail es directa — Railway puede leer el docker-compose.yml existente.
